# Herd Simulation Project

## Два режима распределений
- **empirical**: берём значения из списков, полученных из датасета (с повторами). Это повторяет гистограммы.
- **theoretical**: строим теоретические распределения по параметрам датасета (mean/var/min/max):
  - возраст 1-го осеменения: LogNormal
  - сервис-период: LogNormal
  - осеменение->запуск: смесь (пик >=200 + равномерный хвост слева)

## Покупки (3 режима)
- manual
- auto_counter (balance, закупка 1-го числа)
- auto_forecast (C3, прогноз дойных на today+lead_time, закупка 1-го числа)

## Запись метрик
record_monthly=True — метрики пишутся только 1-го числа месяца (симуляция идёт по дням).

## Производительность
- record_monthly=True уменьшает память и ускоряет агрегацию.
- n_jobs > 1 распараллеливает Monte-Carlo по прогонам (multiprocessing spawn-safe).
